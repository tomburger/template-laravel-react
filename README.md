# Laravel + React Headless Template

Headless Laravel API + React TypeScript frontend, hosted from a single app root in `src`.

## Project Structure

```
template-laravel-react/
├── start.cmd                    # Starts Laravel + Vite dev servers
├── deploy/                      # Deployment scripts (Node)
├── src/                         # Laravel application root
│   ├── app/
│   ├── routes/
│   ├── database/
│   ├── frontend/                # React source (TSX, pages, components)
│   ├── resources/views/         # Blade shell for Vite assets
│   ├── public/build/            # Vite production output
│   ├── package.json             # Frontend toolchain scripts
│   └── composer.json            # Backend dependencies
└── .copilot-instructions.md
```

## Quick Start

### 1. Install Dependencies

```bash
cd src
composer install
npm install
```

### 2. Configure Environment

```bash
cd src
copy .env.example .env
php artisan key:generate
php artisan migrate
```

If you are on macOS/Linux, use `cp .env.example .env` instead of `copy`.

### 3. Run Development Servers

Option A (recommended on Windows):

```bash
.\start.cmd
```

Option B (manual):

```bash
cd src
php artisan serve
npm run dev
```

App URLs:
- Laravel API: `http://localhost:8000`
- Frontend (Vite): `http://localhost:5173`

## Development Notes

- Local development uses Vite live watching (`npm run dev`).
- Do not run `npm run build` for routine checks unless a production build is explicitly needed.
- Regenerate API docs when backend endpoints change: `cd src && php artisan scribe:generate --no-interaction --force`.

## Architecture

- Backend: Laravel REST API with token auth
- Frontend: React 18 + TypeScript + Vite + Bootstrap
- API client/codegen: Orval config in `src/frontend/orval.config.ts`
- Build integration: Laravel serves assets generated by Vite into `src/public/build`

## Deployment

CI/CD workflow file:
- `.github/workflows/deploy-sftp.yml`

Current workflow behavior:
- Builds from `src`
- Generates Scribe docs
- Runs Vite production build
- Stages `src` as deploy payload (with excludes)
- Uploads files through `deploy/deploy.js`
- Executes remote Laravel post-deploy commands (migrate, optimize)

Required secrets used by the workflow:
- `SFTP_PASSWORD`
- `SSH_PRIVATE_KEY`
